O que é uma Branch

Uma branch é uma extensão do projeto inicial. Ela pode ser uma nova funcionalidade, atualização, correção de erro ou qualquer item que precise ser desenvolvido separadamente do projeto principal, evitando que o site sofra alterações constantes enquanto está no ar. Um projeto que não exige essa cautela pode dispensar o uso de branches, caso atualizações frequentes e diretas não causem problemas. Um bom exemplo de implementação de branches ocorre no desenvolvimento em equipe, onde cada integrante desenvolve uma parte do site separadamente para que, ao final, o código seja mesclado à branch master ou main. Todas as branches são criadas a partir da branch principal.


Como fazer uma branch

Primeiro, deve-se criar um repositório. Em seguida, desenvolve-se a branch main ou master; isso é necessário porque qualquer branch posterior será apenas uma adição ou ramificação da branch principal.

Passo a Passo no GitHub desktop para fazer uma Branch: Current branch → New branch → Coloca o nome da branch → Create branch → Publish branch.


Hospedagem

Para conseguir hospedar um site no GitHub Pages, é necessário que seu site esteja público e não seja comercial.


Como hospedar um site

Passo a Passo para hospedar um site no GitHub Pages: Selecione o repositório → Settings → Role até GitHub Pages e clique em "Check it out here!" → Vá em Source e mude de None para Main → Salve as configurações clicando em Save. Você receberá um link com seu site já hospedado. Veja-o no GitHub Pages.

Obervação: Você deve deixar o index.html na raiz do repositório, sem colocá-lo em nenhuma pasta. Se ele for colocado em alguma pasta, o sistema não conseguirá encontrá-lo e apenas hospedará o README.
